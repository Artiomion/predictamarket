## Слайд 3 — Архитектура модели

**Temporal Fusion Transformer (pytorch-forecasting)**


| Параметр                      | Значение                                                               |
| ----------------------------- | ---------------------------------------------------------------------- |
| Параметров                    | **16.27M**                                                             |
| hidden_size / attention_heads | 256 / 4                                                                |
| Encoder length                | 60 торговых дней (~3 месяца lookback)                                  |
| Prediction length             | 22 торговых дня (~1 месяц forecast)                                    |
| Output                        | 7 квантилей (0.02, 0.10, 0.25, **0.50 median**, 0.75, 0.90, 0.98)      |
| Loss                          | QuantileLoss (модель учится предсказывать **распределение**, не точку) |
| Нормализация                  | GroupNormalizer + softplus per-ticker                                  |


**Почему TFT:**

- Attention-механизм выделяет важные факторы → есть Variable Importance
- Квантильный output = **доверительные интервалы из коробки** (80% CI, 95% CI)
- Multi-horizon один forward pass → прогноз на 1d/3d/1w/2w/1m одновременно
hidden_size = 256

Внутри модели каждая "штука" (день, тикер, прогноз) = вектор из 256 чисел. Больше размер = модель умнее, но медленнее. 256 — средне, подобрано под наши данные.
  attention_heads = 4
Модель смотрит на историю через 4 разных "взгляда" одновременно. Один видит краткосрок, другой — тренды, третий — макро, четвёртый — волатильность. Потом все мнения складываются.

**Что тут на что идёт — по блокам:**

### 1. Входной конвейер (0.4M параметров)

**Технически:**

- `input_embeddings` (18.6K) — encode категориальных переменных: ticker (400 штук), sector (12), day_of_week, month. Каждая категория → вектор размера 256.
- `prescalers` (28.7K) — linear layers для нормализации непрерывных входов перед остальной сетью.

**🗣 Простыми словами:** Этот блок превращает всё в числовые векторы одного формата. Размер небольшой, но без него модель не запустится — это как адаптер к разъёму.

### 2. Variable Selection Networks — **ядро TFT** (13M параметров, 80% модели)

**Технически:**

- `static_variable_selection` (305K) — решает какие **статические** переменные (ticker, sector) важны для каждого конкретного прогноза
- `encoder_variable_selection` **(12.6M)** 🎯 —  Это attention-like механизм, который решает "насколько эта фича важна сейчас". Отсюда родом Variable Importance который мы показываем в UI waterfall chart.
- `decoder_variable_selection` (103K) — аналогично но на известных будущих фичах (day_of_week будущих 22 дней)

### 3. Static context GRNs (1.05M параметров)

**Технически:**

- Четыре `static_context`_* GatedResidualNetwork (263K каждый) — берут инфу о статических переменных (ticker, sector) и «раздают» её LSTM'ам и другим блокам как bias. Фактически модель **помнит что AAPL — Tech, а XOM — Energy** сквозь всю сеть.

**🗣 Простыми словами:** если убрать этот блок — модель будет прогнозировать "среднюю акцию", не зная в контексте какой конкретно. Эти 4 блока раздают "паспорт тикера" (ticker + sector embedding) во все остальные части модели как фоновую информацию.

Название "**Gated** Residual Network" — значит в блоке есть механизм "не передавать" информацию если она сейчас не нужна. Позволяет модели гибко включать/выключать контекст.

### 4. Temporal backbone — LSTM (1.05M параметров)

- **Encoder** — читает 60 дней в прошлое, постепенно понимая "о, цена падала, потом развернулась, volume вырос, RSI был в зоне перепроданности"
- **Decoder** — на основе этого понимания генерирует 22 дня в будущее день за днём

Почему LSTM а не популярный ныне Transformer? Для нашей задачи — **60 дней истории это мало**, не нужна сложная attention-механика которая "стоит" квадратично от длины. LSTM дешевле, обучается быстрее, и на 60 временных шагах работает не хуже Transformer.

**Gate поверх LSTM** — это ещё один "выключатель": если LSTM выдал бесполезный сигнал на этом конкретном прогнозе, gate разрешит модели его игнорировать и передать вверх по сети сырой вход. Полезно когда история очень шумная (например после корпоративного события).

### 5. Attention блок (0.7M параметров) — источник интерпретируемости

**Технически:**

- `static_enrichment` (329K) — смешивает static context с временным представлением
- `**multihead_attn` (164K)** 🎯 — **Interpretable Multi-Head Attention (4 головы)**. Именно из attention weights этого блока TFT показывает "какие дни в прошлом повлияли на прогноз". Обычный Transformer attention размазан — TFT-вариант специально сделан так, чтобы веса можно было интерпретировать.
- `post_attn_gate_norm` (132K) — очередной GateAddNorm (TFT любит gates)

**🗣 Простыми словами:** внимание (attention) — это механизм "на какие дни в прошлом модель смотрит больше всего когда делает прогноз". Как если бы перед принятием решения трейдер выделил маркером 5-10 ключевых дней в истории: "вот earnings день, вот начало rally, вот пробой уровня".

**4 головы (heads)** — это как 4 разных эксперта смотрят на одни и те же данные параллельно, каждый со своей "точки зрения". Один обращает внимание на краткосрочные паттерны, второй на долгосрочные тренды, третий на volume, четвёртый на взаимодействия с макро. Их выводы объединяются.

**Чем отличается от обычного Transformer attention:**  TFT-вариант специально сделан так, что attention weights **можно извлечь и интерпретировать**. Это и есть источник объяснимости модели: на странице тикера ты видишь "модель уделила больше всего внимания 3 дням назад и 20 дням назад" — это реальные числа из этого блока.

### 6. Output head (0.4M параметров)

**🗣 Простыми словами:** после того как вся сеть переварила вход и контекст, на выходе остаётся **256-мерный вектор "понимания"** для каждого из 22 будущих дней. Последний крохотный слой (всего 1792 параметра!) превращает этот вектор в **7 чисел**: нижний край 95% диапазона, нижний край 80%, нижний 50%, **медиана**, верхний 50%, верхний 80%, верхний 95%.

**Ключевые архитектурные решения и почему:**


| Решение                                                                                          | Почему                                                                                                                                                    |
| ------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Все 107 фичей идут в **общую** VariableSelectionNetwork, а не отдельные encoder'ы на модальность | Позволяет модели учить **cross-modal interactions** (news sentiment + EPS surprise + VIX одновременно влияет на price)                                    |
| LSTM а не Transformer encoder                                                                    | Для финансовых временных рядов с длинным фиксированным lookback LSTM дешевле и без проблем с quadratic attention cost. 60 × 60 attention matrix не нужен. |
| 4 головы attention (не 8 или 16)                                                                 | `hidden_size=256 / 4 heads = 64 per head` — sweet spot: достаточно heads для diverse patterns, достаточно dimensions per head чтобы не было нойза         |
| `GroupNormalizer` per-ticker                                                                     | Каждый тикер нормализуется **по своей** цене. Price movement в доле, не в долларах. NVDA и KO сравнимы                                                    |
| Softplus transformation                                                                          | Ensures positive predictions. Цена не может быть отрицательной                                                                                            |
| Quantile output вместо point prediction                                                          | Даёт CI 80/95% **без Monte Carlo и без отдельной variance head** — естественно выходит из QuantileLoss                                                    |

## Слайд 4a — Как новости превращаются в числа: FinBERT + PCA

### Шаг 1: FinBERT → 768-мерный вектор на статью

**FinBERT** — это BERT (Bidirectional Encoder Representations from Transformers) от Google, специально **дообученный на финансовых текстах**: отчётности, новости Bloomberg/Reuters, call-транскрипты. Разработан ProsusAI, open-source на HuggingFace.

**🗣 Простыми словами:** BERT — это огромная нейросеть (110M параметров) которая умеет "понимать" текст. Получает предложение → выдаёт **768-мерный вектор** (768 чисел), который содержит "смысл" этого предложения. Если два предложения имеют близкое значение — их векторы окажутся рядом в 768-мерном пространстве. FinBERT именно этому научен на финансовой лексике.

Плюс мы берём **sentiment score** (positive / neutral / negative с confidence 0-1) — это отдельный head FinBERT, который напрямую классифицирует тональность. Этот score мы показываем пользователю на странице `/news` рядом с каждой статьёй.

### Шаг 3: PCA сжимает 768 → 32 размерности

**PCA (Principal Component Analysis)** — алгоритм линейного сжатия. Смотрит на данные в высокомерном пространстве и находит **32 ортогональные оси** (principal components), которые объясняют максимум дисперсии.

**🗣 Простыми словами:** представь облако точек в 768-мерном пространстве. PCA находит "главные направления" этого облака — как если бы ты стоял над облаком и выбирал на какую ось смотреть чтобы увидеть максимум разнообразия. Берём 32 самых "содержательных" направления и **каждый 768-мерный вектор FinBERT → 32 числа по этим направлениям**.

**Важные детали реализации:**

- PCA обучили **один раз** на всём корпусе новостей 2000-2024 → сохранили в `models/pca_model.pkl` (2 MB)
- Используем `IncrementalPCA` из scikit-learn — работает батчами, не требует загрузки всего в память
- Выбор 32 компонент — эмпирический (пробовали 16, 32, 64, 128): 32 дают ~85% объяснённой дисперсии, дальше marginal gain
- В TFT эти 32 становятся фичами `sent_0`..`sent_31` + дополнительно `news_count` (сколько статей в день для тикера) = итого **33 фичи из категории Sentiment**

### Шаг 4: Как это попадает в прогноз

**Daily aggregation:** если за день вышло 5 новостей о тикере — берём FinBERT по каждой, усредняем 768-мерные векторы, прогоняем через PCA → получаем один 32-мерный вектор на день на тикер. Добавляем к остальным фичам.

## Слайд 6 — Процесс обучения

**Инфраструктура:** Google Colab Pro · A100 40GB GPU · ~14 часов total · ~$10 compute

**Гиперпараметры:**

- Batch size 128, lr 3e-4, ReduceLROnPlateau (patience=4)
- Gradient clip 0.1, dropout 0.1
- 7 эпох сохранено (`save_top_k=-1` — ВСЕ чекпоинты для ensemble study)
- EarlyStopping patience=5 на val_loss

**Критический урок и как его исправили:**

EarlyStopping выбрал epoch 1 (val_loss 7.32) как "best", но на реальном test это **худшая** эпоха по MAPE. Причина: `predict=True` в валидации давал всего 400 samples → шумная метрика.

**Что сделали чтобы не довериться этой ошибке:**


| #   | Действие                                                              | Результат                                                                             |
| --- | --------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| 1   | `save_top_k=-1` → сохранили **все 7 чекпоинтов** вместо одного "best" | 1.4 GB на диске как страховка от неверного выбора EarlyStopping                       |
| 2   | Проигнорировали ep1 ("best" по val_loss)                              | В production взяли **ep5** — на основе реальных торговых метрик                       |
| 3   | Отдельная честная оценка на **9 200 sliding windows** (ноутбук 04)    | Полный test вместо 400 samples → MAPE/Sharpe/WR по каждой эпохе (таблица на слайде 7) |
| 4   | Ensemble study на этой честной оценке (ноутбук 06)                    | Per-surface веса: ep5-heavy для Top Picks, ep2-heavy для Alpha Signals                |
| 5   | Temporal robustness testing (ноутбук 09)                              | Подтвердили что ep5 даёт Sharpe > 0 на 5/5 недельных суб-окон                         |


**Как поправить на уровне самого обучения в v2** (код готов в обновлённом ноутбуке 03, запуск в roadmap):

- `predict=False` в validation → ~35 000 sliding windows вместо 400 → val_loss станет стабильной метрикой
- Кастомный Lightning callback после каждой эпохи считает Top-20 Sharpe → EarlyStopping оптимизирует **напрямую торговую метрику**, а не proxy

**Итог:** baseline-результаты которые вы видите на слайдах 7-9 — **это уже постфактум честная оценка**, а не то что "сказал" EarlyStopping. Проблема идентифицирована, обойдена в текущей версии, и будет полноценно исправлена в retrain v2.

### Что каждая метрика означает (кратко)

**Top-20 %** — доходность стратегии "каждый день покупаем 20 тикеров с лучшим прогнозом, держим 1 день, ребаланс" за 23 торговых дня back-test. Чем больше, тем больше заработали. Сравниваем с S&P 500 buy-and-hold = +7.73%.

**Top-20 Sharpe** — главная метрика качества стратегии. Показывает сколько доходности получаем **на единицу риска** (волатильности). 0.5 — уровень S&P 500, 1.0-2.0 — уровень хедж-фондов. У ep5 = 1.36 → попадаем в hedge-fund диапазон.

**ConfLong Sharpe** — Sharpe узкой стратегии "покупаем **только когда модель супер-уверена**" (нижний край 80% CI выше текущей цены). Очень избирательный фильтр даёт высокие цифры на малой выборке — поэтому в live ждём 1.3, а не 5-8.

**ConfLong N** — сколько таких супер-уверенных сделок модель нашла за 23 дня. Разброс от 5 до 513 между эпохами. Маленькое N → цифры шумные, большое N → модель слишком "разборчива" и теряет фильтр.

**ConfLong WR (Win Rate)** — процент прибыльных сделок в ConfLong. 50% = случайность, 60%+ = реальный edge на отобранных сигналах. ep2 = 61.1%, ensemble вытягивает до 64.3%.

**MAPE 1d (Mean Absolute Percentage Error)** — средняя ошибка прогноза цены на **следующий день**, в процентах. 4.74% значит: сказали "$100 завтра" — реально будет $95-105. Для акций это нормально; naive baseline "цена не меняется" даёт 1.9% — поэтому MAPE **не наша сила**.

**DirAcc 22d (Directional Accuracy на 22 дня)** — процент случаев когда модель правильно угадала **направление** (вверх/вниз) на месяц вперёд. 50% = монетка, 52% = едва лучше случайности. DirAcc в продукте **не маркетим** — это слабая сторона.

**Общий паттерн:** разные эпохи лучшие в разном — ep5 по ранжированию, ep2 по conviction, ep4 по короткой точности. Отсюда идея не выбирать одну, а **комбинировать ансамблем** (слайд 8).

## Слайд 9 — Temporal Robustness (NB09) — действительно ли edge стабильный? (устойчивость во времен)

### Зачем это делалось (проблема)

У нас back-test всего **23 торговых дня** (ноябрь 2025 — апрель 2026, 5 месяцев с перекрытием). Это единственное окно, один рыночный режим. Возможно модель "поймала удачу" именно на этом конкретном промежутке — а на другом бы не сработала. Классическая проблема любого ML для финансов: **хороший результат на тесте может быть случайностью**.

Чтобы проверить это, мы разбили test-окно на **5 недельных суб-окон** (каждая по 5 торговых дней, почти не пересекающихся) и прогнали все 7 чекпоинтов на каждом. Если модель даёт положительный edge на **всех 5 неделях подряд** — это уже не случайность, это стабильный паттерн.

### Результат: Top-20 Sharpe по каждой неделе


| Epoch    | Неделя 1  | Неделя 2 | Неделя 3  | Неделя 4  | Неделя 5  | Положительных |
| -------- | --------- | -------- | --------- | --------- | --------- | ------------- |
| ep 0     | 1.58      | 1.73     | **−1.14** | **−2.31** | 1.52      | 3/5           |
| ep 1     | 0.25      | 4.25     | **−1.06** | **−0.60** | **−0.32** | 2/5           |
| ep 2     | 2.41      | 3.02     | 0.14      | **−0.95** | 1.38      | 4/5           |
| ep 3     | **−1.63** | 3.08     | **−0.85** | 0.40      | 1.27      | 3/5           |
| ep 4     | 1.24      | 1.08     | 1.00      | **−1.59** | 2.52      | 4/5           |
| **ep 5** | **1.82**  | **1.47** | **0.64**  | **0.21**  | **2.33**  | **5/5** ✅     |
| ep 6     | **−0.07** | 0.51     | 2.16      | **−1.69** | 4.15      | 3/5           |


**Важная находка:** только **ep5** положительна на всех 5 неделях. Остальные эпохи дают "хороший средний Sharpe", но теряют деньги на 1-3 неделях из 5. В реальной торговле это значит максимальная просадка была бы значительно глубже.

### Robustness ranking (с учётом стабильности, не только среднего)(рейтинг по устойчивости)

Взяли формулу `robustness_score = mean(Sharpe) − 0.5 × std(Sharpe)`. Это штрафует модели у которых хоть и высокое среднее, но болтанка между неделями большая.


| Место | Epoch    | Среднее Sharpe | Std Sharpe               | Минимум Sharpe              | Robustness Score |
| ----- | -------- | -------------- | ------------------------ | --------------------------- | ---------------- |
| 🥇    | **ep 5** | 1.29           | **0.77** (самый низкий!) | **0.21** (единственный > 0) | **0.91**         |
| 🥈    | ep 2     | 1.20           | 1.45                     | −0.95                       | 0.47             |
| 🥉    | ep 4     | 0.85           | 1.34                     | −1.59                       | 0.18             |
| 4     | ep 6     | 1.01           | 1.99                     | −1.69                       | 0.02             |
| 5     | ep 3     | 0.45           | 1.65                     | −1.63                       | −0.37            |
| 6     | ep 1     | 0.50           | 1.92                     | −1.06                       | −0.46            |
| 7     | ep 0     | 0.28           | 1.68                     | −2.31                       | −0.56            |


**Объяснение простым языком:** ep5 не самая прибыльная **в среднем** (1.29), но она единственная где **не было плохих недель**. В трейдинге это важнее чем "пиковая" доходность — инвестор не выдержит месяц с −2.3 Sharpe и уйдёт ещё до того как придёт прибыльный период.

### Bootstrap CI — насколько цифрам можно верить?(доверительный интервал через bootstrap".)

Сделали 1000 bootstrap-итераций: случайно пересэмплировали сделки (с возвращением) и пересчитали Sharpe каждый раз. Получили распределение возможных значений:

```
Top-20 Sharpe (Ep 5):
  Точечная оценка:   1.36
  95% CI:            [0.53, 2.10]   ← истинное значение с 95% вероятностью внутри
  Медиана bootstrap: 1.41
  P(Sharpe > 0):     99.5%   ← почти точно положительный, не случайность
  P(Sharpe > 1.0):   86.7%   ← с вероятностью 87% выше 1.0 = сильный edge
```

**Что это значит простым языком:**

- **P(Sharpe > 0) = 99.5%** — в 995 случаях из 1000 при пересэмплировании Sharpe оставался положительным. Мы **статистически уверены** что модель работает лучше чем случайность.
- **P(Sharpe > 1.0) = 86.7%** — в 867 из 1000 случаев Sharpe превышал 1.0 (hedge-fund порог). То есть "у нас хороший edge" — не маркетинговая фраза, а вероятность.
- **95% CI [0.53, 2.10]** — истинное значение Sharpe почти наверняка в этом диапазоне. Даже нижняя граница (0.53) — выше чем у S&P 500 (~0.5). Хуже рынка не будет даже в худшем раскладе.

### MAPE стабильность — точность прогноза по цене

На каждой из 5 недель подсчитали MAPE 1d (ошибка прогноза цены на 1 день):

```
Ep 5 MAPE 1d: [4.79%, 4.96%, 4.79%, 4.83%, 4.79%]
Std между неделями: 0.07%
```

Практически одинаковая точность на всех 5 неделях. Модель не "теряется" на отдельных окнах — точность прогноза стабильна независимо от рыночных условий этой недели.

## Слайд 10 — Что модель НЕ умеет (честно)

Честный список границ — что **не используем** в продукте:

1. **DirAcc 1d = 48%** — направление на 1 день не угадываем (хуже монетки). Сигналов на intraday/swing нет.
2. **DirAcc 22d = 52%** — на месяц вперёд едва лучше случайности. В маркетинге не упоминаем.
3. **Шорты теряют деньги** — Bottom-20 Short убыточен на всех 7 эпохах. В UI только BUY и AVOID, SELL-сигналов нет.
4. **Bearish bias** — модель в среднем предсказывает падение. ~71% сигналов — SELL/AVOID. В Top Picks фильтруем только BUY с положительным прогнозом.
5. **Sentiment под-утилизирован** — 32 FinBERT-фичи не попали в топ-20 VI. Модель учится по OHLCV/SEC, не по новостям.
6. **Single test window** — edge проверен на 5 месяцах одного режима, не на crash/bear/боковик.
7. **28 тикеров заблокированы** — post-split data mismatch (NVDA, AVGO, WMT, GS, GE...). 346 → 318 активных.

Все эти каветы **обозначены в UI** как caveat-тултипы на каждой headline-метрике. Пользователь видит границы сразу, а не узнаёт о них после потери денег.

## Слайд 10a — Что бы я сделал иначе, если бы начинал обучение с нуля

Самая честная форма roadmap — не "что добавить", а "что переделать чтобы не получить текущие каветы". Вот 5 ключевых решений с которых я бы начал v2:

### 1. Все данные из одного источника — split-adjusted с самого начала

**Сейчас:** train на raw ценах из HuggingFace dataset + live на split-adjusted yfinance = mismatch, 28 тикеров заблокированы.

**В v2:** выкидываю HuggingFace, скачиваю всю историю 2000-2025 из yfinance с `auto_adjust=True`. Один формат везде. Blocklist исчезает автоматически — **318 тикеров → 346**.

### 2. Валидация на полном test-наборе, не на одном окне на тикер

**Сейчас:** `predict=True` в validation → 400 samples → шумный val_loss → EarlyStopping выбирает ep1 как "best", хотя реально лучшая ep5. Пришлось сохранять все 7 чекпоинтов и делать пост-анализ вручную.

**В v2:** `predict=False` → ~35 000 sliding windows в валидации + кастомный Lightning callback который после каждой эпохи считает **Top-20 Sharpe** (а не только val_loss) и EarlyStopping оптимизирует прямо торговую метрику. Правильная эпоха выбирается автоматически.

### 3. Ансамбль из разных seeds, а не разных эпох

**Сейчас:** ensemble (ep2+ep4+ep5) — это одна и та же модель на разных эпохах. Residual correlation 0.98. Diversification bonus минимален.

**В v2:** 3 независимых train'а с разными random seeds + возможно с разными hidden_size (256/384/512). Correlation должна упасть до 0.6-0.7. Теория ансамблей говорит что при такой correlation прирост Sharpe ~+0.2-0.4. То есть **с back-test 1.49 → ~1.7-1.9; live target с 1.0 → ~1.3**.

### 4. Sentiment как end-to-end embedding, а не PCA-bottleneck

**Сейчас:** 768-мерный FinBERT embedding сжимается через PCA до 32 компонент, теряя нюансы. Ни один sent_i не попал в top-20 Variable Importance — модель новости игнорирует.

**В v2:** выкидываю PCA. Вместо этого добавляю обучаемый **sentiment embedding layer** прямо внутри TFT — он учится сжимать 768d до 32-64d **под задачу прогноза цены**, а не под сохранение языкового смысла (что делает PCA). Плюс явный флаг `has_breaking_news = |sentiment| > 0.9 AND news_count == 1` для изолированных событий. Ожидание: sentiment входит в top-10 VI, модель реагирует на новости.

### 5. Market regime features для устранения bearish bias

**Сейчас:** mean predicted 1d = −3.5% при реальном +0.1% → 71% сигналов SELL. Модель впитала bearish склонность из обучающего периода 2020-2024.

**В v2:** добавляю явные **regime features** как входы: SPY 20d trend (растущий/падающий рынок), VIX level (спокойный/паника), сектор-ротация momentum, yield curve spread. Модель увидит "**сейчас** растущий рынок" и перестанет применять bearish prior из памяти. Ожидаемое распределение сигналов 50/50 BUY/AVOID вместо 25/71.

---

### Walk-forward валидация — бонус к этим 5 решениям

После того как модель переобучена с этими 5 фиксами — прогоняю её через **walk-forward validation** на 25 лет истории (train 2000→T, test T→T+6m, shift by 3m, repeat). Получаю метрики на bull markets, 2008 GFC, COVID 2020, rate hikes 2022, боковики. Тогда cavet "single window" закрыт окончательно.

### Итого: компактный v2


| Этап                       | Compute       | Время         |
| -------------------------- | ------------- | ------------- |
| Retrain с 5 фиксами        | $10           | 2 недели      |
| Walk-forward валидация     | $120-200      | 1-2 месяца    |
| **Итого до production v2** | **~$150-250** | **~2 месяца** |


**Ключевое:** это не "1000 улучшений", а **5 конкретных изменений в pipeline** которые бьют по главным проблемам сразу. Остальные идеи (Russell 2000, short-side модель, day-trade specialised head) — это v3+, уже после того как v2 докажет edge в walk-forward.

**Главный вывод:** большинство текущих каветов — не фундаментальная проблема TFT или подхода. Это **конкретные решения на этапе подготовки данных и валидации** которые я бы сегодня сделал иначе. Модель правильная, pipeline требует доработки.

- `signal` + `confidence` для каждой строки — модель оценила и подписала каждый тикер
- `predicted_return_1m` как основной критерий сортировки вместо market cap / алфавита — **каталог сразу упорядочен по перспективности**
- `variable_importance` раскрывается по клику (быстрый explainer)

**Как применять для торговли:**

1. **Быстрый screening:** фильтр "signal=BUY + sector=Technology" → список из 318 сразу сжимается до 5-10 идей
2. **Альтернатива биржевому скринеру:** P/E + market cap заменены на AI-predicted return
3. **Earnings awareness:** видишь кто отчитывается → закрыть/уменьшить позицию перед отчётом если не уверен

**Полезность:** заменяет ~3 разных инструмента (Yahoo screener + Finviz + earnings calendar) → **5-10 минут утреннего обзора вместо 30**.

## Слайд 14 — Страница тикера `/stocks/[ticker]` (главный аналитический экран)

**Самая нагруженная страница. Всё что нужно по одной акции — в одном скролле.**

### Блок 1: График + AI-прогноз overlay

- Candlestick свечи (TradingView) с таймфреймами 1D/1W/1M/3M/1Y/5Y и индикаторами SMA/EMA/Bollinger/RSI/MACD
- **Поверх рисуется прогнозная линия модели** — median из `full_curve` (22 торговых дня вперёд)
- **Лента 80% CI** (между q0.10 и q0.90) вокруг линии — плотная
- **Лента 95% CI** (между q0.02 и q0.98) — полупрозрачная, шире
- Hover на любую точку прогноза → popup с median + нижним/верхним краем диапазона

**Какая часть модели:** полный `forecast` dict из API — 5 горизонтов × 7 квантилей.

**Польза для трейдера:** видит не "вырастет до $250", а "**80% что в диапазоне $220-$275, median $248**". Узкий CI = модель уверена, широкий = избегай больших позиций.

### Блок 2: Signal + Confidence badges

- Большой цветной badge: **BUY / HOLD / AVOID** (жёлтый/зелёный/красный)
- Confidence: **HIGH / MEDIUM / LOW** (зависит от того, насколько CI не пересекает current_close)

**Какая часть модели:** `signal` (median vs current_close) и `confidence` (ширина `lower_80`/`upper_80`).

### Блок 3: Rank tier

- **Место тикера в универсуме 318** по predicted_return_1m: "Top 5% / Top 10% / Top 20% / средний / нижний квартиль"
- Отображается как прогресс-бар или тир (🥇🥈🥉) — сразу видно "насколько он хорош на фоне остальных"

**Какая часть модели:** `GET /forecast/{ticker}/rank` — SQL окно с RANK() over всеми 318 тикерами.

**Почему это критично:** MAPE 22d = 12.6% — dollar target ненадёжен ("цена будет $280" может ошибаться на $30). Но **ranking задача** — это то что модель делает хорошо (Sharpe 1.49). Rank tier — правильный способ думать о прогнозе.

### Блок 4: Variable Importance waterfall

- Топ-5 факторов которые **увеличили** или **снизили** прогноз: напр. "Low (−6.2%) bearish, oil (−4.6%) bearish, eps_surprise_pct (+2%) bullish"
- Визуально: horizontal bar chart с цветом по направлению

**Какая часть модели:** TFT attention weights извлечённые во время inference → `variable_importance.top_factors`.

**Польза:** трейдер не слепо доверяет черному ящику. Видит "модель дала BUY потому что inventory уменьшился + insider_buys". Если согласен с логикой → входит. Если нет (напр. верит что oil вырастет, а модель считает oil медвежьим для этой акции) — может переспорить модель и пройти мимо.

### Блок 5: Extreme-forecast warning chip

- Появляется когда `|predicted_return_1m| > 30%` (напр. модель сказала "вырастет на +60% за месяц")
- Chip: "⚠️ Экстремальный прогноз. Используй rank tier, не дольный таргет — MAPE 22d 12.6% на таких значениях станет ~$20-30 ошибки"

**Польза:** защищает от катастрофических сайзингов когда модель шумит.

### Блок 6: Фундаментал / SEC / новости / insiders

- Key metrics (Revenue, EPS, FCF, Profit Margin) с 8Q sparklines
- SEC Edgar Income/Balance/Cash Flow (Pro+)
- News feed по тикеру с FinBERT sentiment
- Earnings history с beat/miss badges
- Insider transactions последние 10

**Как это всё применять для одной сделки:**

1. Открыл тикер с Dashboard (он был BUY + HIGH conf)
2. Посмотрел CI — узкий, модель уверена
3. Посмотрел rank — Top 10% в универсуме
4. Посмотрел Variable Importance — логика соответствует моему тезису
5. Проверил earnings — не ближайшие 10 дней (не хочу сидеть через отчёт)
6. Глянул News sentiment — не было обвальных новостей
7. Вошёл в позицию через своего брокера с размером что диктует confidence (HIGH → полный сайз, MEDIUM → половина)

[Скриншот: `/stocks/AAPL` или другой liquid тикер — вся страница в скролле]

**Какая часть модели:**

**Главная стратегия зарабатывания (passive rebalance):**

5. **Ребалансируешь ежемесячно** — смотришь новый Top-20, выходишь из тех кого там больше нет, докупаешь новичков

**Реалистичные ожидания:**

- Live target alpha ~+4 п.п. / месяц vs S&P 500 (после shrinkage с +12 back-test)

**Затраты vs доход:** Pro $15/мес = $180/год. Чтобы "окупить" надо +$180 дополнительно к S&P 500 → портфель **$4 500** при live alpha 4% в год. Начиная от $5k-10k капитала — математика выгодная.

## Слайд 16 — Alpha Signals `/alpha-signals` (Premium)

**Для более серьёзного трейдера. $39/мес оправдывается для капитала от $20-30k.**

**Что показывает:**

- Лента только тех сигналов где **все 3 модели ensemble согласны** что `lower_80 > current_close` (то есть даже нижний край 80% CI выше текущей цены — супер-узкий "уверенный BUY")
- Для каждого: disagreement score (насколько модели разошлись по median), breakdown трёх прогнозов, predicted return
- Sort по predicted_return_1d (краткосрочные "очевидные" сделки вверху)

**Какая часть модели:**

- **ep2-heavy ensemble [0.5/0.3/0.2]** — максимум Win Rate (64.3% back-test, ~56% live target)
- Фильтр `confident_long == True` (берётся из поля БД где AND 3 моделей lower_80 > current_close)
- Disagreement score — stddev между quantile forecasts 3 моделей

**Чем это отличается от Top Picks:**

- Top Picks — **ranking**, покупаешь много позиций с предположением что в среднем они обгонят рынок
- Alpha Signals — **conviction**, покупаешь мало позиций, но в каждую вкладываешься уверенно

**Стратегия зарабатывания (conviction trades):**

1. Подписан на `/alpha-signals` — приходит WebSocket-алерт когда появляется новый consensus BUY
2. Берёшь **только consensus сигналы** — в среднем их ~5-15 в неделю (редкие!)
3. **Размер позиции 2-3× обычного** — модель уверена, ты увереннее
4. **Holding period 5-22 дня** (тот горизонт на котором модель прогнозирует)
5. Stop-loss ниже `lower_80` — если пробило нижний край CI, модель неправа, выход

**Реалистично:**

- Win Rate ~56% на 20-40 сделок в год = ~12-22 победных из 20-40
- Sharpe target ~1.3 live (в back-test был 8.04, но там выборка 28 — overfit)
- Больше effort (следить за алертами, активный sizing), больше потенциальный доход

## Слайд 19 — News `/news` (контекст к сигналам)

**Что показывает:**

- Агрегированная лента RSS: Yahoo Finance, Reuters, MarketWatch, Bloomberg, CNBC
- Каждая новость с **FinBERT sentiment** — positive / neutral / negative + confidence score
- Фильтры: по тикеру, сектору, sentiment, дате
- Badge "High Impact" когда sentiment score превышает порог (алерт по таким)

**Какая часть модели:**

- FinBERT embeddings (768d) → PCA 32d попадают в training как фичи модели TFT (`sent_0..sent_31`)
- На Fronted показывается raw sentiment — не из TFT, а отдельный FinBERT forward pass на каждую новость

**Caveat который я объявляю открыто:** Variable Importance показывает что PCA-компоненты **не попали в топ-20 важнейших фичей модели**. Модель фактически **плохо учится по новостям**. Причины — sparse news coverage + короткий train на sentiment + агрессивное PCA. Retrain v2 с расширенными источниками и меньшей PCA-компрессией должен это исправить.

**Как применять пока:**

- News **как human-readable sanity check**. Модель дала BUY → открываешь News по тикеру → не было ли обвальных новостей за ночь (банкротство, SEC investigation, CEO departure)
- Если есть свежая negative news которую модель не "увидела" (ей нужно время на FinBERT + следующий inference) — ты можешь принять решение раньше и не входить

**Полезность:** защита от "модель знает за вчера, ты знаешь за сегодня".

## Приложение C — Ожидаемые вопросы и ответы

**Q: Почему Sharpe live target 1.0, а в back-test 1.49?**
A: Single-window test 23 дня → overfit возможен. Shrinkage heuristic: live target = ~70% of back-test. Честнее обещать меньше и превзойти.

**Q: Модель работает в кризис / в 2008?**
A: Training включает 2000-2024 (dot-com bust, 2008 GFC, COVID 2020). В памяти модели эти режимы есть. Но test на одном окне 2025-2026 — не проверили явно на crash window. Walk-forward validation закроет этот вопрос.

**Q: Почему только 318 тикеров, а не все 500?**
A: 346 = пересечение текущего S&P 500 с 400 обученных. Минус 28 из blocklist (post-split mismatch). После retrain на split-adjusted prices восстановим все 346+.

**Q: Почему не GPT/LLM?**
A: LLM хороши в генерации текста, но плохи в multi-horizon time series с quantile output. TFT — специализированная архитектура для этой задачи. 16.3M params (не 70B) = дешевле inference.

**Q: Что с SEC Edgar coverage?**
A: ~~88-91% на AAPL/NVDA/MSFT. Некоторые XBRL концепты не у всех компаний (~~12 из 27 полей). Не критично для прогноза — остальные фичи компенсируют.

**Q: Монетизация реалистична при таких метриках?**
A: $15/мес за 10 прогнозов + Top-20 Pro — конкурирует с Seeking Alpha Premium ($239/год), Zacks Premium ($249/год). Нужно всего ~100 подписчиков для самоокупаемости хостинга. Fair pricing.
